# 2 多线程通信和同步

## 2.1 多线程状态

### 2.1.1 线程状态说明

- 初始化（Init）：该线程正在被创建。
- 就绪（Ready）：该线程在就绪列表中，等待CPU调度。
- 运行（Running）：该线程正在运行。
- 阻塞（Blocked）：该线程被阻塞挂起。Blocked状态包括：pend（锁、事件、信号量等阻塞）、suspend（主动pend）、delay（延时阻塞）、pendtime（因为锁、事件、信号量等超时等待）。
- 退出（Exit）：该线程运行结束，等待父线程回收其控制块资源。

### 2.1.2 竞争状态和临界区

竞争状态（Race Condition）：多线程同时读写共享数据。

临界区（Critical Section）：读写共享数据的代码片段

避免竞争状态策略，对临界区进行保护，同时只能有一个线程进入临界区。
